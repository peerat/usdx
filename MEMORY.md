# uSDX memory and module map

Цель этого файла - держать рядом с прошивкой практическую карту модулей:
что за что отвечает, какие функции можно отключать для ATmega328P, и сколько
это дает flash/RAM по текущим сборкам PlatformIO.

## Базовая сборка

Команда:

```bash
platformio run -e uno
```

Текущий результат baseline:

| Память | Использовано | Всего | Свободно |
| --- | ---: | ---: | ---: |
| RAM | 1440 B | 2048 B | 608 B |
| Flash | 31504 B | 32256 B | 752 B |

Это уже рабочая зона, но запас flash очень маленький: меньше 1 KB. Поэтому
любые новые команды CAT, строки меню или DSP-функции надо сразу проверять
сборкой.

## Профили PlatformIO

`platformio.ini` содержит базовый профиль `uno` и набор измерительных профилей
`uno_no_*`. Каждый `uno_no_*` отключает ровно один модуль через `build_flags`,
чтобы увидеть его цену.

```bash
platformio run -e uno
platformio run -e uno_no_swr
platformio run -e uno_no_cw_decoder
platformio run -e uno_no_cat
```

Эти профили не обязательно являются рекомендуемыми прошивками для радио. Это
измерительный инструмент: сравниваем с baseline и видим, какой модуль сколько
стоит.

## Экономика памяти

Baseline до группировки меню: RAM 1390 B, Flash 30214 B.

Текущий baseline форка после правок меню, экрана, адаптивного `ScanStop`,
отключения VOX и Semi-QSK по умолчанию и перевода `QUAD` в compile-time опцию:
RAM 1440 B, Flash 31504 B.

| Профиль | Что отключено | RAM | Flash | Экономия RAM | Экономия Flash |
| --- | --- | ---: | ---: | ---: | ---: |
| `uno_no_cat` | CAT-интерфейс | 1244 B | 28888 B | 196 B | 2616 B |
| `uno_no_cw_decoder` | CW-декодер | 1371 B | 29402 B | 69 B | 2102 B |
| `uno_no_swr` | SWR-метр | 1358 B | 29898 B | 82 B | 1606 B |
| `uno_no_keyer` | CW keyer | 1398 B | 30456 B | 42 B | 1048 B |
| `uno_no_rit` | RIT | 1440 B | 31144 B | 0 B | 360 B |
| `uno_no_lpf` | LPF switching | 1440 B | 31162 B | 0 B | 342 B |
| `uno_no_vox` | VOX | 1440 B | 31504 B | 0 B | 0 B |
| `uno_no_semi_qsk` | Semi-QSK | 1440 B | 31504 B | 0 B | 0 B |

Вывод по экономии:

1. Самый дорогой модуль - CAT: примерно 2.6 KB flash и 196 B RAM.
2. Если CAT нужен, самые крупные кандидаты на отключение - CW-декодер и SWR.
3. Keyer тоже заметный, но его нельзя отключать, если нужна нормальная CW-работа.
4. RIT и LPF дают меньше, но могут добрать сотни байт, когда сборка упирается в лимит.
5. `uno_no_vox` и `uno_no_semi_qsk` сейчас ничего не выигрывают, потому что эти
   функции уже выключены в текущем baseline.
6. Экономию нельзя строго складывать арифметически: линкер удаляет код блоками,
   и комбинации флагов могут дать немного другой результат.

## Модули

### CAT

Отвечает за управление трансивером по последовательному интерфейсу:
частота, режим, PTT, RIT/XIT-похожие команды, запрос состояния и совместимость
с частью Kenwood-подобных команд. Основной разбор идет через `analyseCATcmd()`.

Текущие особенности:

- буфер команды ограничен `CATCMD_SIZE = 16`;
- переполнение буфера отвечает `E;` и сбрасывает прием команды;
- частота печатается без `sprintf()`, чтобы не тащить лишний форматтер;
- команды вида `AI`, `AG`, `XT`, `RT`, `RS`, `FL`, `VX` сейчас в основном
  дают фиксированные ответы или подтверждают принятое значение.

Цена: около 2616 B flash и 196 B RAM. Отключается профилем `uno_no_cat`.

### SWR meter

Отвечает за измерение и отображение КСВ через мост на A6/A7. Используется в TX
связанных путях и на LCD-экране защиты/индикации. Для текущего baseline это
один из самых дорогих необязательных модулей.

Цена: около 1606 B flash и 82 B RAM. Отключается профилем `uno_no_swr`.

### CW decoder

Декодирует CW на приеме и обновляет экран декодера. Модуль удобный, но дорогой:
у него есть состояние декодирования, буферы и логика обновления LCD.

Цена: около 2102 B flash и 69 B RAM. Отключается профилем
`uno_no_cw_decoder`.

### CW keyer

Обрабатывает телеграфный ключ, режимы iambic/single, скорость WPM и часть CW TX
логики. Это не то же самое, что CW-декодер: keyer нужен для передачи CW.

Цена: около 1048 B flash и 42 B RAM. Отключается профилем `uno_no_keyer`.

### RIT

Дает приемный сдвиг частоты и связанную CAT-команду установки смещения. Влияет
на отображение VFO и быстрый пересчет частоты Si5351.

Цена: около 360 B flash, RAM почти не меняется. Отключается профилем
`uno_no_rit`.

### VOX

Автоматически включает передачу по уровню аудио. Использует состояние сэмплов,
порог и логику входа/выхода из TX.

В текущем baseline модуль уже выключен в конфигурации, поэтому профиль
`uno_no_vox` не дает дополнительной экономии. Если вернуть `VOX_ENABLE`, это
будет один из средних по цене необязательных модулей.

### LPF switching

Управляет переключением НЧ-фильтров по диапазону. В текущей конфигурации активен
вариант DL2MAN uSDX Rev3 через I2C GPIO-расширитель.

Цена: около 342 B flash, RAM почти не меняется. Отключается профилем
`uno_no_lpf`.

### Semi-QSK

Держит прием приглушенным короткое время после манипуляции, чтобы CW-переходы
не звучали рваными. Небольшой модуль, но при жестком лимите flash тоже считается.

В текущем baseline модуль уже выключен в конфигурации, поэтому профиль
`uno_no_semi_qsk` не дает дополнительной экономии.

### DSP/RX/TX ядро

Это основа прошивки: обработка I/Q, фильтры, AGC/NR, S-meter, генерация частоты,
переход RX/TX, PWM TX и обслуживание таймингов. Этот код почти не является
опциональным: экономить память здесь опаснее, потому что легко сломать звук,
частоту дискретизации или тайминги прерываний.

### LCD/OLED и UI

Текущий код поддерживает LCD и OLED-варианты. `OLED` включается только если
заданы `OLED_SSD1306` или `OLED_SH1106`. Меню, курсор шага частоты, баннеры,
индикаторы режима и S-meter находятся в UI-части и тесно связаны с обработкой
энкодера/кнопок.

В этом форке меню сгруппировано в разделы `General`, `RX`, `TX`, `CW`.
Группировка удобнее для пользователя, но занимает дополнительный flash, поэтому
новые пункты меню нужно добавлять особенно осторожно.

`CW_DEBUG_DISPLAY` отключен: в CW-режиме экран не должен перекрываться
диагностическими таймингами. Включать его стоит только временно для отладки
декодера.

`ScanStop` добавлен в RX-меню и использует уже существующий расчет уровня
S-meter/dBm. Остановка адаптивная: за проход запоминается минимальный уровень,
а настройка задает превышение над этим минимумом. Срабатывание подтверждается
коротким наблюдением, затем частота удерживается около 10 секунд и скан
продолжается, если пользователь ничего не нажал. Отдельная DSP-цепочка для
остановки скана не добавлялась, чтобы не расходовать flash.

## Что сейчас фактически включено

По активным `#define` в текущей конфигурации видны:

- модель `RED_CORNERS`;
- `SWR_METER`;
- `LPF_SWITCHING_DL2MAN_USDX_REV3`;
- `CAT` и `CAT_FAST`;
- `KEYER`;
- `CW_DECODER`;
- `TX_ENABLE`;
- `RIT_ENABLE`;
- `PTX`, `CAL_IQ`.

По умолчанию сейчас выключены:

- `VOX_ENABLE`;
- `SEMI_QSK`;
- `QUAD`.

Важно: простой поиск по тексту может ловить `#define` внутри закомментированных
блоков. Для окончательной проверки всегда использовать сборку конкретного
профиля.

## Текущие предупреждения ревью

- В файле много старых повторных `#define`. Это исторический стиль выбора
  конфигурации, но он шумит в сборке и повышает риск, что реально активное
  значение окажется не тем, которое видно первым.
- `freeMemory()` сейчас не вызывается, но оставлен как якорь для генерации
  прототипов Arduino/PlatformIO и как ручной отладочный инструмент RAM.
- Предупреждение `operation on 'ea1' may be undefined` было вызвано двойным
  присваиванием вокруг statement-макроса `EA()`; такие вызовы заменены на
  прямой `EA(...)` без внешнего `=`.
- В профильных сборках могут появляться `unused` предупреждения. Это нормально
  для измерительных профилей, но если профиль станет постоянной прошивкой, такие
  переменные лучше закрывать тем же `#ifdef`, что и отключаемый модуль.

## Практическая стратегия экономии

Для прошивки с CAT сначала смотреть на отключение CW-декодера или SWR:

- без CW-декодера: примерно 29.2 KB flash;
- без SWR: примерно 29.5 KB flash;
- без keyer: примерно 30.0 KB flash, но CW TX пострадает;
- без CAT: примерно 28.7 KB flash, но теряется управление с компьютера.

Если цель - оставить CAT и получить запас под фиксы, самый разумный порядок:

1. отключить CW-декодер, если прием CW текстом не нужен;
2. отключить SWR, если в железе нет нужного моста или измерение не используется;
3. отключить keyer только если CW TX не нужен;
4. добирать мелочь через RIT/LPF по реальному использованию радио.
